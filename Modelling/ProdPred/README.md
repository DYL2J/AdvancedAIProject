# Re-order Logistic Model

This section provides the basics of a Logistic model for predicting re orders for different users for the Advanced AI module

# Structure

- `dataCreation.py`  
  Generates a synthetic grocery dataset with realistic purchasing and reordering behaviour.  
  Outputs the CSV files listed below.

- `users.csv`  
  Contains user identifiers (200 users).

- `products.csv`  
  Contains product identifiers and product categories (150 products).

- `orders.csv`  
  Contains order-level information, including user ID and order number.

- `order_products.csv`  
  Contains product-level order data, including the binary `reordered` label.

- `reorder_model.joblib`  
  Trained sklearn pipeline including preprocessing and logistic regression classifier.

- `product_purchase_counts.joblib`  
  Dictionary mapping product IDs to total purchase counts.

- `product_reorder_rates.joblib`  
  Dictionary mapping product IDs to average reorder rates.

These artefacts allow inference without retraining the model.

# Data

Data was generated synthetically based on the *Instacart* Dataset which can be found here:

For more information on how data was created, refer to:

There are 150 products for users to order from, which can be assigned names later. Currently given IDs 1-150

Target Variable to predict:
reordered

if the user re orders an item from a previous cart, this is set to 1.
